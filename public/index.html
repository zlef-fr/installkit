Why InstallKit

Most install prompts are wrong half the time

Generic “Add to Home Screen” banners show iPhone steps to Android users and vice-versa. InstallKit gets the details right — including the cases other widgets ignore.

Coverage

The exact path, per platform

A native prompt when the browser supports it; the real, current manual steps when it doesn’t; and an honest redirect when installing simply isn’t possible.

Integration

Drop in, then tune it

Works as-is with one tag. Configure it with data-attributes or the JS API.

recommended — capture the prompt early
<!-- 1 · capture Chrome's install event before our script loads -->
<script>window.__ik=window.__ik||{};addEventListener('beforeinstallprompt',e=>{e.preventDefault();__ik.deferred=e},{once:true});</script>

data-* attributes

AttributeDefaultWhat it does
data-app-name—Your app's name, used throughout the copy.
data-icon—URL of a square icon shown in the sheet header.
data-positionbottom-rightLauncher position: bottom-right / -left / -center / inline.
data-accentoliveAccent colour (any CSS colour) to match your brand.
data-themeautoauto / dark / light.
data-modeautoauto shows a launcher; manual = you call InstallKit.open().
data-delay1200ms before the launcher appears.
data-remind-after3Days to stay hidden after a visitor dismisses it.
data-analyticsfalseOpt-in anonymous funnel (counts only, no PII).

JavaScript API

InstallKit.open()Open the install sheet on demand (e.g. from your own button).
InstallKit.canInstall()true unless already installed or unsupported.
InstallKit.env()The full detection object (os, browser, method…).
InstallKit.on(evt, fn)Events: ready, open, close, choice, installed, dismiss.
InstallKit.config({…})Override config at runtime.
